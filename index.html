Mini Project 2: Basic Calculator

Task 1: Develop a web-based calculator with computation history

Requirements:

  • Single HTML file (index.html)
  • External CSS and JavaScript files
  • No inline or embedded CSS/JS
  • No external libraries
  • No use of eval()

Functionality:

  • Basic arithmetic operations (+, -, ×, ÷)
  • Accept input via buttons or typing (numbers and decimals only)
  • Record computation history when:
    • Pressing the "=" button
    • Pressing an operation button during a longer calculation
  • Clickable history list to reuse previous results in new calculations
  • "C" button to reset calculator state and clear errors
  • "Clear History" button to delete all history entries
  • Display "Error" for invalid operations (e.g., divide by zero); user must reset with "C"

Design:

  • Polished layout using CSS (flexbox or grid; no tables)
  • Restrict typed input to valid characters (numbers, decimals, and operations only)
  • Handle all basic calculations as per provided test cases

A few Cases:

  • 111 + 000 = 111
  • 0.5 + 0.6 = 1.1
  • 111 + - 111 = 0
  • 2 × 13 = 26
  • 30 ÷ 10 = 3

    Project Name: Mini Project 4: Howler

    Description: Create a client-side rendered social network app that allows users to post "howls," view other users' howls, and manage follows/unfollows.

    Features:

    • Users can post text entries ("howls") to share thoughts.
    • View howls from followed users on the main page, sorted in reverse chronological order.
    • Visit user profiles to view their howls and manage follow/unfollow status.
    • Basic authentication to identify the current user.

    Data:

    • Users: Each user has a unique ID, username, first and last name, and an avatar URL.
    • Howls: Each howl includes an ID, user ID, timestamp, and text content.
    • Follows: Tracks which users are followed by others.

    Backend Requirements:

    • Design and implement a REST API using Express.
    • API routes must be mounted under /api.
    • Data is stored in memory and loaded from provided JSON files.
    • Endpoints include:
      • Authenticating a user.
      • Fetching user data and howls.
      • Posting howls.
      • Following/unfollowing users.
    • Use express.Router() for modular routes.

    Frontend Requirements:

    • Responsive UI using Bootstrap for styling.
    • Dynamic rendering of data fetched from the REST API.
    • Frontend includes:
      • Login Page: Simple login mechanism to identify the user.
      • Main Page: Displays howls from the logged-in user and followed users.
      • User Profile Page: Displays user details, howls, and follow/unfollow functionality.
