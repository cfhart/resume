EatSafe Progressive Web Application (PWA)


Duration: August - December 2024


Description

EatSafe is a Progressive Web Application designed to help users find restaurants in their area that accommodate various food allergies, dietary preferences, and restrictions. The app aims to provide a seamless experience for users seeking safe dining options that align with their specific needs.

Main Features


Home Page

  • Displays a picture and logo
  • Provides options to log in or sign up

Search Page

  • Allows users to find restaurants based on location, allergens, and dietary preferences
  • Includes filtering and sorting options for restaurants
  • Has a regular search bar for users to search by restaurant name

Profile Page

  • Displays user's first name, last name, and username
  • Allows users to update their dietary preferences, allergies, and location
  • Includes options to update the user's avatar and view saved favorite restaurants

User Data & Authentication

  • User passwords must include a symbol and be at least 8 characters long upon creation
  • Password data is stored as hashed and salted for security
  • Users can log in, log out, and sign up
  • Session tokens are used for authentication and authorization
  • If session expires or authentication fails, the user cannot access profile data

Restaurant Data

  • Each restaurant has the following attributes: name, location, rating, allergens, diets, and phone number
  • Restaurants can be filtered by allergens (e.g., milk, peanuts), dietary preferences (e.g., kosher, vegan), and location
  • Each restaurant can be added to or removed from the user's favorites

PWA Capabilities

  • The app works offline, caching GET requests to ensure users can still access previously loaded data
  • New restaurants or updated data will not be available offline until accessed at least once
  • Login and user creation are not available offline, but users can still search for cached restaurants

Frontend Development

  • Designed the HTML and CSS for the home, login, sign-up, and search/filter pages.
  • Implemented sorting and filtering functionality using JavaScript.
  • Created a responsive and accessible user interface.

Backend Development

  • Developed REST API endpoints to handle user authentication and data retrieval.
  • Set up backend routes for managing user accounts and restaurant data.
  • Integrated a relational database (MariaDB) to store user information and restaurant details.

Future Enhancements

  • Implement push notifications for restaurant updates.
  • Enable offline data input and synchronization when back online.
  • Integrate persistent storage using IndexedDB for improved offline support.

  • Frontend includes:
    • Login Page: Simple login mechanism to identify the user.
